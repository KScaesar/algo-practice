## 回溯問題的兩種核心思維模式

「回溯三問」，以「構造子集」為例，說明如何將問題拆解為遞迴結構：

* 當前操作？
* 子問題？
* 下一個子問題？

https://www.bilibili.com/video/BV1mG4y1A7Gu/?vd_source=deac8b505de301c629b87c1abebac9f3

---

### 方法1.站在「輸入」的角度思考：選或不選 (Input Perspective)

這種做法核心在於對原數組中的**每個元素**進行二進制式的抉擇。

* **核心操作：** 每一層遞迴對應原數組的一個下標 i。
* **決策分支：**

  * **不選：** 直接跳過第 i 個數，進入 i+1 的決策。
  * **選：** 將第 i 個數加入路徑，進入 i+1 的決策，完成後回溯（移除該數）。
* **答案產出：** 當遞迴處理完所有元素（下標達到數組長度 n）時，**葉子節點**即為答案。
* **回溯三問：**

  * **當前操作：** 枚舉第 i 個數「選」或「不選」。
  * **子問題：** 從下標 ≥ i 的數字中構造子集。
  * **下一個子問題：** 進入下標 ≥ i+1 的決策。

🔑 **順序意圖補充：**

> 此方法天然遵守「index 單調遞增」，因此生成的是**不考慮順序的子集合**
> （不會出現 [2,1]，只會有 [1,2]）

---

### 方法2.站在「答案」的角度思考：枚舉元素 (Result Perspective)

這種做法的核心思維：**「下一個加入的元素要選誰？」**
你站在目前的子集（path）上，往後看從 index i 開始的所有候選元素，逐一嘗試把每個當作「下一個加入的元素」。

* **核心操作：** 使用一個 `for` 迴圈枚舉從當前下標 i（**含 i 本身**）開始到結尾的所有可能數字。
* **決策分支：**

  * 選取下標 j（j ≥ i）的元素加入路徑。
  * ⚠️ **不是為了避免重複，而是為了維持順序約束：**
    下一個選取的元素下標必須從 j+1 開始，確保 index 單調遞增。
* **答案產出：** 遞迴樹的**每一個節點**（包括空集）都是一個合法的子集。
* **回溯三問：**

  * **當前操作：** 從下標 j ≥ i 中枚舉一個數字加入路徑。
  * **子問題：** 從下標 ≥ i 的數字中構造子集。
  * **下一個子問題：** 從下標 ≥ j+1 的數字中繼續構造。

🔑 **順序意圖補充：**

> 此方法本質仍然維持「index 單調遞增」，因此：
>
> * [1,2] 會出現
> * [2,1] 不會出現
>
> 👉 重點不是「避免重複」，而是**不允許改變原始順序**

## 兩種方法的答案產出時機對比

以 `[1, 2, 3]` 求所有子集為例，說明「答案在哪個節點被收集」。

---

### 方法1：答案在**葉子節點**

每一層對一個元素做「選 or 不選」，**一定要走完所有層**才算完成一個子集。

```
                      []              ← 根，還沒決定完
              /               \
       不選1 []              選1 [1]
           /    \            /      \
     不選2 []  選2 [2]  不選2 [1]  選2 [1,2]
        /\      /\       /\          /\
       /  \    /  \     /  \        /  \
      []  [3] [2] [2,3][1] [1,3]  [1,2][1,2,3]
      ↑    ↑   ↑    ↑   ↑    ↑     ↑     ↑
                    葉子節點 → 才收集答案 ✅
```

```python
def dfs(i, path):
    if i == n:          # 走到底 → 才收集
        res.append(path[:])
        return
    # 不選
    dfs(i+1, path)
    # 選
    path.append(nums[i])
    dfs(i+1, path)
    path.pop()
```

---

### 方法2：答案在**每一個節點**

核心問題：**「下一個加入的元素選誰？」**
從 index i（含 i 本身）之後，逐一嘗試每個候選。**進入每個節點時，路徑本身就是合法子集，立刻收集。**

```
              []              ← 根，也是答案 ✅ (空集)
           /  |   \
   選1:[1]  選2:[2]  選3:[3]   ← for j in range(0,3)，各自是答案 ✅
         / \       \
選2:[1,2] 選3:[1,3]  選3:[2,3]  ← [3] 無候選，停止
        |
      選3:[1,2,3]              ← 再無候選，停止
```

> 每條**邊**代表「選了誰作為下一個」，每個**節點**就是一個合法子集。

```python
def dfs(i, path):
    res.append(path[:])   # 進來就收集，不管在哪一層
    for j in range(i, n): # 從 i 開始（含 i），不是 i 之後
        path.append(nums[j])
        dfs(j+1, path)
        path.pop()
```

---

### 對比總結

| | **每個節點在問的問題** | 樹的形狀 | 答案產出位置 |
|---|---|---|---|
| **方法1（Input）** | 「第 i 個元素，**選 or 不選**？」 | 完美二叉樹 | 葉子節點 |
| **方法2（Result）** | 「下一個加入的元素，**選誰**？」 | 不對稱多叉樹 | 每個節點 |

> 兩個方法最終產出的子集完全相同（`2^n` 個），只是**思考角度**、**樹的形狀**和**收集時機**不同。
