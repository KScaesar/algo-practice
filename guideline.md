## 白板解題核心流程

每題都強制：
1. 先問清楚
2. 寫例子
3. 列完整輸出
4. 找規律
5. 才寫 code

### 1.先確認題目
- 輸入 / 輸出是什麼? 型別，長度範圍
- 「分組 / 子集合」的定義? 是否連續？ 是否可重複？ 順序重要嗎(排列/組合)？
- 是要「列出結果」還是「只算數量」

### 2.用小例子列出所有輸出
- 用具體資料「反推問題結構」
- 自己舉 2 個最小案例（n=2,3）
- 手動列出「全部可能結果」
- 不要急著想解法

### 3.找規律
- 有幾個決策點
    - 選 / 不選，決定「要不要加入」(子集合，順序不重要)
    - 切 / 不切，決定「要不要切斷」(連續區間)
    - 這個位置放誰，決定「要放誰」(順序重要)
- 輸出數量變化是否有公式（`2^n`、`2^(n-1)`、`C(n,k)`、`n!`）
- 是否可拆成子問題

### 4.選解法
- 列出所有 → Backtracking
- 算數量 → Math / DP
- 有重複子問題 → DP
- 有規律 → 直接公式
